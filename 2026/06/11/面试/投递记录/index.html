百度 · LLM RL Infra

  • 岗位：RL Infra（文心基模团队）
  • 进度：1面 ✅ → 2面 ✅ → 等待结果
  • 整体感受：1&2 面连面，偏 infra 深度，问题基本围绕 RL 训练框架展开。面试官对 verl 很熟悉，建议提前把源码读透。

面经总结（高频题 / 知识点）

ZeRO 1/2/3 分片策略

Stage 分片内容 新增通信
ZeRO-1 Optimizer states reduce-scatter（optimizer step 后）
ZeRO-2 Optimizer states + Gradients + all-gather（backward 中）
ZeRO-3 Optimizer states + Gradients + Parameters + all-gather（forward 中）
  • ZeRO vs TP：ZeRO 是数据并行维度的内存优化，parameter 在每个 GPU 上完整计算后释放；TP 是模型并行，parameter 在 GPU 间真正拆开同时计算。两者可以组合使用。

verl remove_padding

  • 将 batch 内所有序列 flatten 成 1D，用 cu_seqlens 记录边界
  • flash attention 直接接受 varlen 输入，避免对 padding token 做无效计算
  • 收益最大场景：序列长度分布不均匀时（如 RLHF rollout 阶段生成长度差异大）
  • remove_padding vs sequence packing：前者去掉单条序列内的 padding；后者把多条短序列拼成一条长序列，两者可叠加

MoE 训推不一致

  1. Expert 并行分组不同：训练和推理的 EP group 不一样，同一 token 被路由到不同物理 GPU
  2. auxiliary loss 副作用：训练时负载均衡 loss 影响路由分布，推理时无此 loss，分布可能偏移
  3. top-k capacity 限制：训练时有 expert capacity buffer（丢弃溢出 token），推理时通常不丢

Tensor / Sequence Parallelism

  • TP：将单个矩阵乘法按列/行切分到多卡（Megatron ColumnParallelLinear / RowParallelLinear），forward 需 all-reduce
  • SP：序列维度切分到多卡，减少 activation 显存，通常和 TP 配合；layernorm/dropout 在 SP 维度，attention 在 TP 维度
  • SP vs ring attention：SP 是 TP 组内的 activation 切分；ring attention 是跨更大 context 的 attention 并行

Python 单例

class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

线程安全版需加锁，先在锁外判断一次，进锁后再判断一次（double-checked locking）。

LeetCode 88 原地合并有序数组

从后往前双指针，避免覆盖 nums1 中还未比较的元素。

def merge(nums1: list, m: int, nums2: list, n: int) -> None:
    i, j, k = m - 1, n - 1, m + n - 1
    while i >= 0 and j >= 0:
        if nums1[i] > nums2[j]:
            nums1[k] = nums1[i]; i -= 1
        else:
            nums1[k] = nums2[j]; j -= 1
        k -= 1
    while j >= 0:
        nums1[k] = nums2[j]; j -= 1; k -= 1
