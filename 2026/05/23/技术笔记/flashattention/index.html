FlashAttention

算法 笔记 infra
技术笔记
Publish Date:   2026-05-23

标准 attention 是 memory-bound：算力闲着，卡在 HBM 反复读写 S、P 两个 N×N 大矩阵上。FA 没有把它变成 compute-bound，而是把内存墙推开——降 HBM 流量，让这个 memory-bound 算子尽量贴近算力上限。它的全部魔法只有一个母版递推：“老的所有 × 修正项 + 现在的”。online softmax 把 3-pass 压成 2-pass，FA 把 2-pass 压成 1-pass，后面 tiling、IO、显存全是把这套递推搬到 GPU 上。

一、大前提:Online Softmax(3-pass → 2-pass)

一句话：每次 m 变化，分母 d 就是"原来的所有 × 修正项 + 现在的"。这是 FA 全部递推的母版。

标准 safe softmax 必须扫 3 遍:① 求最大值 m;② 求分母 d;③ 算输出 y。每次 pass 都要把 x 从 HBM 再读一遍。

online softmax 的洞察:扫到第 i 个元素时,同时维护"到目前为止"的 m 和 d。代价是 d 的更新多了一个修正项 exp(m_{i-1} − m_i),把"旧的累加值"换基准到新的 m 上。

di′←di−1′⋅emi−1−mi+exi−mid_i' \leftarrow d_{i-1}' \cdot e^{m_{i-1} - m_i} + e^{x_i - m_i} di′​←di−1′​⋅emi−1​−mi​+exi​−mi​

  • m 没变 → 修正项 = exp(0) = 1 → 普通累加;
  • m 变大 → 修正项 < 1 → 把"旧的所有"等比例缩小,数学严格等价(不是近似)。

它是 FA 的"前提"，不是"灵感"。 softmax 天然要整行的全局 max、sum。没有这种"边走边维护"的递推，你就必须把整行 S 一次性看到，也就必须落 HBM。FA 能跨块合并 softmax，合法性完全来自它。

二、FA 总结:1-pass + O 修正系数怎么来的

把 online softmax 搬到 attention 上是 2-pass(先维护 (m,d),再算 a 乘 V 得 O)。但 a 不是终点，O 才是——把 O 的累加也并进第一个 pass,就一次扫完:

di′←di−1′⋅emi−1−mi+exi−mid_i' \leftarrow d_{i-1}' \cdot e^{m_{i-1}-m_i} + e^{x_i - m_i} di′​←di−1′​⋅emi−1​−mi​+exi​−mi​

oi′←oi−1′⋅di−1′⋅emi−1−midi′+exi−midi′V[i,:]\boldsymbol{o}_i' \leftarrow \boldsymbol{o}_{i-1}' \cdot \frac{d_{i-1}' \cdot e^{m_{i-1}-m_i}}{d_i'} + \frac{e^{x_i - m_i}}{d_i'} V[i, :] oi′​←oi−1′​⋅di′​di−1′​⋅emi−1​−mi​​+di′​exi​−mi​​V[i,:]

这个 O 的修正系数是怎么写出来的? 别死记，顺着 O = P·V 推一遍就出来了：

  • O 由 P 乘 V 得到，m 一变，P 就变，所以要修正的本质是 P。
  • P 是个分数 P=ex−mdP = \dfrac{e^{x-m}}{d}P=dex−m​，分子分母都含 m：
    • 分母 d 要换基准 → 乘 exp(m_{i-1}−m_i)（就是第一节那个修正项）；
    • 分子 ex−me^{x-m}ex−m 也要换到新的 m → 同一个修正项。
  • 把这两处修正塞进 O = P·V，就得到上面那条 o 的递推——它就是把 online softmax 的递推线性地搬运到 V 上，形状和 d 的递推完全同构：“老的所有 × 修正项 + 现在的”。

第一、二节是 FA 的命门。后面 tiling、SRAM、IO 全是把这套"逐元素"递推**升级到"逐块"**的工程实现。

三、Forward

3.1 Tiling:逐元素 → 逐块

GPU 整块处理。v1 选"外循环 K/V 块、内循环 Q 块"。外循环固定 K₀，内循环走遍所有 Qᵢ，每个 (Qᵢ, K₀) 算出 Sᵢ₀ → P̃ᵢ₀，顺手更新 Oᵢ₀：

外循环换到 K₁，内循环再遍历 Q，每个 Qᵢ 拿到第二块贡献，跟之前的 Oᵢ₀ 合并（用第二节的修正项递推）：

每个 Q 行块要"等所有外循环走完"才是最终结果。过程中 Oᵢ₀ 只是"看过 K₀ 这些 token"的部分输出，还要把后续 K 块的贡献补进来 → 所以是合并而非覆盖。为什么合并就行？矩阵乘本就是"行·列再求和"，少加几列就少加几项，补回来即可，线性运算不存在冲突。

3.2 Forward 伪代码(看图分析 IO / 循环 / SRAM)

理解 fwd 的一切都要对照伪代码：伪代码告诉你 IO 复杂度、内外循环顺序、SRAM 和 HBM 里分别存什么。

对照上图：

  • 第 1 行:B_c = ⌈M/4d⌉，除以 4 是因为 SRAM 要同时装 Qᵢ, Kⱼ, Vⱼ, Oᵢ 四块(B_r = min(B_c, d))。
  • 外循环 j(K/V)、内循环 i(Q) —— 这个顺序是 v1 一切 IO 行为的根源。
  • 第 9~10 行:Sᵢⱼ、P̃ᵢⱼ 都在 SRAM 算，不落 HBM——省 IO 的根。
  • 第 11~12 行:更新 m, l 和 O(就是第一、二节的递推)。
  • 第 13 行:每轮把 Oᵢ, lᵢ, mᵢ 写回 HBM。

标准 attention forward(对照)：

1. 读 Q,K，算 S = Q·Kᵀ (N×N)，写 S 回 HBM
2. 读 S，算 P = softmax(S) (N×N)，写 P 回 HBM
3. 读 P,V，算 O = P·V (N×d)，写 O 回 HBM

3.3 IO 通信量 & 显存占用(fwd 的核心)

先分清两件常被搅在一起的事：显存占用 = HBM 里同时驻留多大；IO 通信量 = HBM 被累计读写多少次。FA 两个都降，但 O 这一项的 IO 是反向增加的。

逐项看 fwd 把"读写什么"换成了什么：

量 标准 attention FA v1 结论
Q, K, V 各读一遍 N×d tile 反复 load，但所有循环走完恰好各读一遍 持平（都是 O(Nd)）
S, P 写 S + 读 S + 写 P → O(N²) 来回过 HBM 不落 HBM，SRAM 里活一个 Bᵣ×B_c tile 显著降
m, l — 每轮读写，单 tile 只有 Bᵣ×1 新增，但极小
O 最后写一次 N×d 每轮读出加一笔再写回 反向增：Tc−1 次读 + Tc 次写

(m,l) 的维度——必须记牢的易错点。 一句话：(m,l) 只有 i 下标，没有 j。

  • softmax 沿 K 维（每行内部） 归一化。一行 Q 对应一整行 score Si,:S_{i,:}Si,:​，rowmax / rowsum 各塌缩成一个标量 → mi,lim_i, l_imi​,li​ 是 N×1。
  • 那个被求和"吃掉"的维度 j，正是 S/P 从 O(N²) 掉到 (m,l) 的 O(N) 的原因。
  • tile (i,j)(i,j)(i,j) 在 SRAM 里临时算的是局部 m~ij,l~ij\tilde m_{ij}, \tilde l_{ij}m~ij​,l~ij​，但写回 HBM 的是 mi,lim_i, l_imi​,li​——它在每次外循环 j 被原地更新（l_i ← l_i·修正 + l̃_ij），不是每个 j 存一份。所以 HBM 里 l、m 始终只有 N×1。
S, P m, l
完整维度 N×N（带 i,j） N×1（只有 i）
单 tile Bᵣ×B_c Bᵣ×1
能否常驻 SRAM 反复 rescale 不能（N² 装不下，用完即弃） 能（一行一个标量）

神级总结（只看 HBM）：

标准：读 Q/K/V 大矩阵 + 读写 S/P 大矩阵 + 写 O 大矩阵。
FA v1：读 Q/K/V tile + 读写 m/l 小向量 + 反复读写 O tile。
核心变化 = 把 O(N²) 的 S/P 读写换成了 O(N) 的 m/l 读写（主力降），代价是 O 多了 Tc 倍读写（次要增），净降。显存占用同理：S/P 两块 N² 没了，只剩 O(N) 的 m/l。

3.4 为什么 O, m, l 被迫反复写回 HBM?(v1 的命门)

这是最反直觉的点：O、m、l 本来就要反复更新，直接一直摆在 SRAM 里更新不行吗，为什么非写回 HBM？

答案藏在循环顺序里。先看 SRAM 装了什么：按 B_c = ⌈M/4d⌉ 的四等分，SRAM 任意时刻只装得下当前一对 tile 的工作集（K、V、Q、O 各一块，m/l 太小忽略）。

v1 外循环 K₀ 固定，内循环要走遍 所有 Q：处理 Q₁ 时 SRAM 放 O₁；走到 Q₂，O 的位置得腾给 O₂，O₁ 被挤出去。等外循环换 K₁、内循环回到 Q₁，O₁ 早已不在 SRAM，只能从 HBM 读回来。

根本矛盾：

想常驻 SRAM 的 SRAM 实际能装的
v1 所有 Tr 个 Oᵢ, mᵢ, lᵢ（每个都还没算完，每轮都要更新） 只够当前一个 tile 的工作集

Oᵢ, mᵢ, lᵢ 是"每行 Q 各一份"，共 Tr 份、O(N) 级。SRAM 才 ~100KB，装下当前 tile 就满了，不可能把 Tr 行的半成品全焊在 SRAM。所以只能"算一点 → 踢回 HBM → 下轮捞回来"。

一句话点破：不是"O/m/l 能不能常驻 SRAM"，而是 v1 任意时刻有 Tr 个 O 都处于半成品状态，SRAM 装不下这么多半成品，只能寄存 HBM。

钩子 → v2：v2 把外循环换成 Q——一次只让一个 Oᵢ 处于半成品，内循环走遍所有 K 块时它全程待在 SRAM，最后才写一次 HBM。"反复读写 m/l/O"这个毛病是 v2 靠循环顺序消除的，与存什么无关（详见 fa2）。

3.5 IO 与显存复杂度

HBM IO 额外显存
标准 attention O(N²)（S, P 来回过 HBM 主导） O(N²)（S, P 两块矩阵）
FlashAttention v1 O(N²·d²/M)（M = SRAM 容量） O(N)（只多 m, l）

A100 上 M ≈ 100KB、d = 64，FA 的 HBM 读写比标准低约 10×——背后是 SRAM 带宽（~19 TB/s）比 HBM（~1.5 TB/s）快约 10×。"省 IO"不是不读了，是把读的位置从 HBM 换到 SRAM。

易错：FA 的 FLOPs 不省反略多（修正项乘法、末尾除 l_new）。省的纯粹是 HBM 流量——这才是 memory-bound 算子的真瓶颈。

四、Backward

bwd 唯一的核心创新就是 recomputation：前向不存 S、P，反向时在 SRAM 里把它们现场重算一遍。 一切都对照伪代码看：谁在外循环、谁被反复累加、SRAM 里重算了什么。

4.1 Recomputation:重算什么、靠什么重算

bwd 算梯度绕不开 P（O = PV → dV = Pᵀ·dO、dP = dO·Vᵀ …）。标准做法是 fwd 把 P（N×N）存 HBM、bwd 读回；FA 不存 P，bwd 在 SRAM 里逐 tile 现场重算。依据就是 fwd 顺手存下的标量：

Pij=exp⁡(Sij−mi)liP_{ij} = \frac{\exp(S_{ij} - m_i)}{l_i} Pij​=li​exp(Sij​−mi​)​

只要 Q,K 在（Sij=QiKj⊤S_{ij} = Q_i K_j^\topSij​=Qi​Kj⊤​ 能重算）、再加 fwd 存的统计量，P 就能还原。这和 fwd 里 (m,l) 能跨块合并 softmax 是同一个数学事实——统计量是"压缩掉的 N×N 信息的无损钥匙"。

关键工程细节(必记)：bwd 其实只存一个标量，不是两个。 fwd 收尾把 m、l 合成 logsumexp：

Li=mi+log⁡(li)L_i = m_i + \log(l_i) Li​=mi​+log(li​)

bwd 直接 Pij=exp⁡(Sij−Li)P_{ij} = \exp(S_{ij} - L_i)Pij​=exp(Sij​−Li​) 一步还原，不用分别拿 m 和 l。存一个标量比存两个省一半，且还原 P 更直接。

注意别和 3.4 搅在一起：L 是 fwd 算完后压成一个标量落盘给 bwd 用的；3.4 那个"(m,l) 反复读写 HBM"是 fwd 过程中的事、根源是循环顺序。两码事，一个是落盘格式，一个是循环代价。

4.2 bwd 的 IO / 显存(和 fwd 同一个故事)

显存占用：标准要读回 N×N 的 P、还要给 dS/dP 腾 N×N；FA 不存 P，dS/dP 只在 SRAM 活一个 Bᵣ×B_c tile 不落 HBM。常驻 HBM 的只剩 Q,K,V,O,dO,dQ,dK,dV 这些 O(Nd) + L 的 O(N) → 又把 O(N²) 摁回 O(N)。

IO 流量：bwd 外循环 K/V（j）、内循环 Q（i）。用 fwd 那套"看下标定循环"的判断法——

  • dKj,dVjdK_j, dV_jdKj​,dVj​（j 下标，要吃掉所有 i 的贡献）是被反复累加的主角，j 在外循环 → 它们能常驻、逐块累加。
  • dQidQ_idQi​（i 下标）被放在内循环 → 每次外循环换 j 都要把它读出加一笔再写回，反复读写 HBM；同时多个 j 块往同一个 dQidQ_idQi​ 写，需要 atomic add。

"反复读写"和"atomic add"是同一病根的两面：根源都是 i 下标的 dQ 被放在了 j 外循环的内层。

  • 串行视角 → 每轮读出加一笔写回 → 流量代价；
  • 并行视角 → 多个 j 块同时写同一行 → 正确性代价（写冲突）。

钩子 → v2：v2 bwd 把外循环换成 K/V、让 dKj,dVjdK_j,dV_jdKj​,dVj​ 这俩 j 下标主角常驻，dQ 的处理方式随之改变，atomic add 一并消除（和 fwd 循环顺序钩子呼应，详见 fa2）。

4.3 标准 vs FA backward

维度 标准 backward FA backward
是否存 P 是，从 HBM 读 P 否，只存 L = logsumexp，现场重算
dP, dS 存储 写回 HBM（N×N） 只在 SRAM 算，不落 HBM
dK, dV 累加 一次性矩阵乘 外循环 j 中逐块累加
dQ 累加 一次性矩阵乘 内循环 i 中累加，需 atomic add
dropout 重读 P 存 PRNG seed，重生成 mask

重计算的 FLOPs 代价：约多一次 fwd 的量。但 attention 是 memory-bound、FLOPs 占比低，拿算力换带宽——和 fwd 一模一样的 tradeoff，净赚。

dropout 是 recomputation 的延伸：存 mask 是 N×N bit，显存回到 O(N²)；改存一个 PRNG seed（O(1)），bwd 按同 seed 重生成 mask。凡是 O(N²) 的中间量，能重算就不存。
