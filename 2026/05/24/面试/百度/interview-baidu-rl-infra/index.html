4. DeepSpeed ZeRO 1-3

被问到 ZeRO 各阶段的分片策略和通信开销。

Stage 分片内容 新增通信
ZeRO-1 Optimizer states reduce-scatter（optimizer step 后）
ZeRO-2 Optimizer states + Gradients + all-gather（backward 中）
ZeRO-3 Optimizer states + Gradients + Parameters + all-gather（forward 中）

关键追问：

  • ZeRO-3 的 parameter all-gather 在 forward 时发生，会有什么额外开销？
  • 和 tensor parallelism 的区别是什么？ZeRO 分片 vs TP 切片的本质不同？
  • 什么时候选 ZeRO-3，什么时候选 TP？

ZeRO 是数据并行维度的内存优化，parameter 在每个 GPU 上是完整计算后释放；TP 是模型并行，parameter 在 GPU 间真正拆开同时计算。两者可以组合使用。

5. verl 的 remove_padding

verl 在 rollout / training 阶段对变长序列做了 padding 移除优化。

核心思路：

  • 将 batch 内所有序列 flatten 成 1D，用 cu_seqlens（cumulative sequence lengths）记录边界
  • attention kernel（如 flash attention）直接接受 varlen 输入，避免对 padding token 做无效计算
  • 在 loss 计算时也只对有效 token 做 reduce

追问：

  • 这个优化在什么场景下收益最大？（序列长度分布不均匀时，比如 RLHF rollout 阶段生成长度差异大）
  • remove_padding 和 sequence packing 的区别？

remove_padding 是去掉单条序列内的 padding；sequence packing 是把多条短序列拼成一条长序列塞满 context window，两者可以叠加。

6. MoE 训推不一致 R3

MoE（Mixture of Experts）在训练和推理时的路由行为不一致问题，在 verl 这类训推一体框架里尤为关键。

常见不一致来源：

  1. Expert 并行分组不同：训练时 EP group 和推理时不一样，导致同一个 token 被路由到不同物理 GPU 上
  2. 负载均衡 loss 的副作用：训练时 auxiliary loss 会影响路由分布，推理时没有这个 loss，分布可能偏移
  3. top-k 路由的 capacity 限制：训练时有 expert capacity buffer（丢弃溢出 token），推理时通常不丢，行为不同

面试官问的 R3 可能指某个内部框架的版本或 routing 的第三个问题，具体记不清了，大方向是训推一致性的工程保证。

7. Tensor / Sequence Parallelism

Tensor Parallelism（TP）：

  • 将单个矩阵乘法按列/行切分到多卡
  • Megatron-LM 的 ColumnParallelLinear / RowParallelLinear
  • forward 需要 all-reduce（或 all-gather + reduce-scatter）

Sequence Parallelism（SP）：

  • 将序列维度切分到多卡，减少 activation 显存
  • 通常和 TP 配合使用（Megatron-LM 的 SP 模式）
  • layernorm / dropout 在 SP 维度做，attention 在 TP 维度做
  • 需要额外的 all-gather / reduce-scatter 来切换维度

追问：SP 和 ring attention 的区别？

SP 是 Megatron 里 TP 组内的 activation 切分；ring attention 是跨更大 context 的 attention 并行，两者目标相似但实现层次不同。

8. Python 单例

手写单例模式，考察对 Python 元类 / __new__ 的理解。

class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

注意 double-checked locking：先在锁外判断一次，避免每次都抢锁；进锁后再判断一次，防止并发初始化。

9. 算法题：原地合并有序数组

经典题（LeetCode 88），给两个有序数组 nums1（有足够空间）和 nums2，原地合并。

关键：从后往前双指针，避免覆盖 nums1 中还没比较的元素。

def merge(nums1: list, m: int, nums2: list, n: int) -> None:
    i, j, k = m - 1, n - 1, m + n - 1
    while i >= 0 and j >= 0:
        if nums1[i] > nums2[j]:
            nums1[k] = nums1[i]
            i -= 1
        else:
            nums1[k] = nums2[j]
            j -= 1
        k -= 1
    # nums2 有剩余时直接填入（nums1 剩余不用动）
    while j >= 0:
        nums1[k] = nums2[j]
        j -= 1
        k -= 1

时间复杂度 O(m+n)，空间复杂度 O(1)。
