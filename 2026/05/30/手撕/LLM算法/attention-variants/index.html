Attention 变体手撕：MHA / MQA / GQA / MLA

手撕 attention transformer kv-cache
手撕 LLM算法
Publish Date:   2026-05-30

面试高频手撕，Attention 系列从 MHA 到 MQA、GQA，再加 KV Cache，考察对 head 分组、显存节省原理的理解。

MHA（Multi-Head Attention）

import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0
        
        self.d_model = d_model
        self.num_heads = num_heads
        self.head_dim = d_model // num_heads
        
        self.q_linear = nn.Linear(d_model, d_model)
        self.k_linear = nn.Linear(d_model, d_model)
        self.v_linear = nn.Linear(d_model, d_model)
        self.out_linear = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        batch_size, seq_len, d_model = x.size()
        
        # (B, L, D) -> (B, H, L, d_h)
        q = self.q_linear(x).view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
        k = self.k_linear(x).view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
        v = self.v_linear(x).view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)

        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(self.head_dim)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        attn = F.softmax(scores, dim=-1)
        
        # (B, H, L, d_h) -> (B, L, D)
        context = torch.matmul(attn, v).transpose(1, 2).contiguous().view(batch_size, seq_len, d_model)
        return self.out_linear(context)

易错点

  1. 类必须继承 nn.Module，否则参数无法被优化器感知，模型无法训练。
  2. 多头拼接后必须过 out_linear，将 H 个 head 的输出重新映射回 d_model 空间。
  3. forward 要接收并处理 mask 参数，causal mask 不加则 decoder 会看到未来 token。

MHA + KV Cache

Decode 阶段只输入 1 个新 token，K/V 历史拼接复用，避免重复计算。

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0
        self.d_model = d_model
        self.num_heads = num_heads
        self.head_dim = d_model // num_heads
        self.q_linear = nn.Linear(d_model, d_model, bias=False)
        self.k_linear = nn.Linear(d_model, d_model, bias=False)
        self.v_linear = nn.Linear(d_model, d_model, bias=False)
        self.out_linear = nn.Linear(d_model, d_model, bias=False)
    
    def forward(self, x, mask=None, kv_cache=None):
        B, L, D = x.shape
        H, d_h = self.num_heads, self.head_dim
        
        q = self.q_linear(x).view(B, L, H, d_h).transpose(1, 2)
        k_new = self.k_linear(x).view(B, L, H, d_h).transpose(1, 2)
        v_new = self.v_linear(x).view(B, L, H, d_h).transpose(1, 2)
        
        # 拼接历史 KV
        if kv_cache is not None:
            k_cache, v_cache = kv_cache
            k = torch.cat([k_cache, k_new], dim=2)
            v = torch.cat([v_cache, v_new], dim=2)
        else:
            k, v = k_new, v_new
        
        new_kv_cache = (k, v)
        
        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(d_h)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        attn = F.softmax(scores, dim=-1)
        L_q = q.size(-2)
        context = torch.matmul(attn, v).transpose(1, 2).reshape(B, L_q, D)
        
        return self.out_linear(context), new_kv_cache

MHA + RoPE（旋转位置编码）

RoPE 是目前主流大模型（LLaMA / Qwen / DeepSeek）的标配位置编码。用 MHA 承载最简单：和普通 MHA 唯一的区别，就是在「投影出 Q、K」之后、「算注意力」之前，插入对 Q、K 的旋转。x 本身不动（区别于老式 x + PE），不增加任何可学参数。

def apply_rope(x, cos, sin):
    # x: (B, H, L, d_h)；cos/sin: (1,1,L,d_h//2)，按位置预算好，可广播
    x1, x2 = x[..., 0::2], x[..., 1::2]          # 相邻两维拆成一对的两半
    rot_x1 = x1 * cos - x2 * sin                 # 二维旋转公式
    rot_x2 = x1 * sin + x2 * cos
    return torch.stack([rot_x1, rot_x2], dim=-1)  # 交错铺回 d_h

class MHA_RoPE(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = d_model // num_heads
        self.q_linear = nn.Linear(d_model, d_model, bias=False)
        self.k_linear = nn.Linear(d_model, d_model, bias=False)
        self.v_linear = nn.Linear(d_model, d_model, bias=False)
        self.out_linear = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x, cos, sin, mask=None, kv_cache=None):
        B, L, D = x.shape
        H, d_h = self.num_heads, self.head_dim

        # ① 投影：和普通 MHA 完全一样，RoPE 未参与
        q = self.q_linear(x).view(B, L, H, d_h).transpose(1, 2)
        k_new = self.k_linear(x).view(B, L, H, d_h).transpose(1, 2)
        v_new = self.v_linear(x).view(B, L, H, d_h).transpose(1, 2)

        # ② RoPE 夹在这里：只旋转 Q、K，绝不碰 V
        q = apply_rope(q, cos, sin)
        k_new = apply_rope(k_new, cos, sin)

        if kv_cache is not None:                 # 注意：cache 存「旋转后」的 K
            k_cache, v_cache = kv_cache
            k = torch.cat([k_cache, k_new], dim=2)
            v = torch.cat([v_cache, v_new], dim=2)
        else:
            k, v = k_new, v_new
        new_kv_cache = (k, v)

        # ③ 注意力：q·k 这一步，绝对位置被自动消成相对距离
        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(d_h)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        attn = F.softmax(scores, dim=-1)
        context = torch.matmul(attn, v).transpose(1, 2).reshape(B, q.size(-2), D)
        return self.out_linear(context), new_kv_cache

推导一：RoPE 怎么让 token 实现"旋转"

旋转只能发生在二维平面里——你没法对一个 128 维向量说"转 30°"。所以 RoPE 把 d_head 维相邻两维配成一对（d_head/2 对），每一对当成一个二维向量，在自己的平面里转一个角度，最后交错铺回去。整个 token 就完成了"旋转"。

二维旋转公式来自把向量写成极坐标 (rcos⁡φ,rsin⁡φ)(r\cos\varphi, r\sin\varphi)(rcosφ,rsinφ)、再转 θ\thetaθ 角变成 (rcos⁡(φ+θ),rsin⁡(φ+θ))(r\cos(\varphi+\theta), r\sin(\varphi+\theta))(rcos(φ+θ),rsin(φ+θ))，用和角公式展开即得代码那两行。关键性质：旋转只改方向、不改长度——所以位置信息被塞进"朝向"，token 的内容（模长那部分）不被破坏。

推导二：内积为什么把绝对位置变成相对位置

施加旋转时是绝对位置：位置 m 的 q 转 α=mθ\alpha=m\thetaα=mθ，位置 n 的 k 转 β=nθ\beta=n\thetaβ=nθ，各转各的。变相对发生在 scores = q·k 这一步——

把两个旋转后向量的内积展开，交叉相乘项恰好凑成 差角公式 cos⁡αcos⁡β+sin⁡αsin⁡β=cos⁡(α−β)\cos\alpha\cos\beta+\sin\alpha\sin\beta=\cos(\alpha-\beta)cosαcosβ+sinαsinβ=cos(α−β) 和 sin⁡αcos⁡β−cos⁡αsin⁡β=sin⁡(α−β)\sin\alpha\cos\beta-\cos\alpha\sin\beta=\sin(\alpha-\beta)sinαcosβ−cosαsinβ=sin(α−β)。于是结果里角度只剩 α−β=(m−n)θ\alpha-\beta=(m-n)\thetaα−β=(m−n)θ，绝对位置 m、n 在相减中抵消，只留相对距离 m−n。这就是"内积让绝对变相对"的全部机制：唯有 sin/cos 在内积里会触发差角公式，所以 RoPE 偏偏选了旋转。

MQA（Multi-Query Attention）

K/V 只有 1 个 head，显存节省 H 倍，依赖 broadcast 扩展到 H 个 Q。

class MultiQueryAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0
        self.d_model = d_model
        self.num_heads = num_heads
        self.head_dim = d_model // num_heads
        
        self.q_linear = nn.Linear(d_model, d_model, bias=False)
        # K/V 输出维度从 d_model -> head_dim（只有 1 个 head）
        self.k_linear = nn.Linear(d_model, self.head_dim, bias=False)
        self.v_linear = nn.Linear(d_model, self.head_dim, bias=False)
        self.out_linear = nn.Linear(d_model, d_model, bias=False)
    
    def forward(self, x, mask=None, kv_cache=None):
        B, L, D = x.shape
        H, d_h = self.num_heads, self.head_dim
        
        q = self.q_linear(x).view(B, L, H, d_h).transpose(1, 2)  # (B, H, L, d_h)
        k_new = self.k_linear(x).unsqueeze(1)  # (B, 1, L, d_h)
        v_new = self.v_linear(x).unsqueeze(1)
        
        if kv_cache is not None:
            k_cache, v_cache = kv_cache
            k = torch.cat([k_cache, k_new], dim=2)
            v = torch.cat([v_cache, v_new], dim=2)
        else:
            k, v = k_new, v_new
        
        new_kv_cache = (k, v)
        
        # broadcast: k (B,1,L_k,d_h) 自动扩展到 (B,H,L_k,d_h)
        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(d_h)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        attn = F.softmax(scores, dim=-1)
        L_q = q.size(-2)
        context = torch.matmul(attn, v).transpose(1, 2).reshape(B, L_q, D)
        return self.out_linear(context), new_kv_cache

GQA（Grouped-Query Attention）

MHA 和 MQA 的折中：G 组 K/V head（1 < G < H），cache 省 H/G 倍。
关键：cache 存 G-head 版本，attention 计算时用 repeat_interleave 临时扩展到 H。

class GroupedQueryAttention(nn.Module):
    def __init__(self, d_model, num_heads, num_kv_heads):
        super().__init__()
        assert d_model % num_heads == 0
        assert num_heads % num_kv_heads == 0
        
        self.d_model = d_model
        self.num_heads = num_heads
        self.num_kv_heads = num_kv_heads
        self.head_dim = d_model // num_heads
        self.group_size = num_heads // num_kv_heads
        
        self.q_linear = nn.Linear(d_model, d_model, bias=False)
        # K/V 输出维度是 G * head_dim
        self.k_linear = nn.Linear(d_model, num_kv_heads * self.head_dim, bias=False)
        self.v_linear = nn.Linear(d_model, num_kv_heads * self.head_dim, bias=False)
        self.out_linear = nn.Linear(d_model, d_model, bias=False)
    
    def forward(self, x, mask=None, kv_cache=None):
        B, L, D = x.shape
        H, G, d_h = self.num_heads, self.num_kv_heads, self.head_dim
        
        q = self.q_linear(x).view(B, L, H, d_h).transpose(1, 2)       # (B, H, L, d_h)
        k_new = self.k_linear(x).view(B, L, G, d_h).transpose(1, 2)   # (B, G, L, d_h)
        v_new = self.v_linear(x).view(B, L, G, d_h).transpose(1, 2)
        
        if kv_cache is not None:
            k_cache, v_cache = kv_cache
            k = torch.cat([k_cache, k_new], dim=2)
            v = torch.cat([v_cache, v_new], dim=2)
        else:
            k, v = k_new, v_new
        
        new_kv_cache = (k, v)  # cache 存 G-head 版，省显存
        
        # 临时扩展到 H head 用于 attention 计算
        k = k.repeat_interleave(self.group_size, dim=1)  # (B, H, L_k, d_h)
        v = v.repeat_interleave(self.group_size, dim=1)
        
        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(d_h)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        attn = F.softmax(scores, dim=-1)
        L_q = q.size(-2)
        context = torch.matmul(attn, v).transpose(1, 2).reshape(B, L_q, D)
        return self.out_linear(context), new_kv_cache

MLA（Multi-head Latent Attention）

DeepSeek-V2 提出，目的是压缩 KV Cache。GQA 靠"多个 head 共享一份 KV"省显存（损失表达力），MLA 换思路：把 K、V 联合压缩成一个低维 latent c_kv 存进 cache，用时再上投影还原——每个 head 的 KV 仍独立算出，几乎不掉点。

前提概要：为什么需要先懂 RoPE
MLA 推理快的真正来源是「矩阵吸收」：K 的上投影矩阵可提前并入 Q 的投影，推理时不必真的还原 K，直接用 c_kv 算分数。但 RoPE 要求对 K 逐位置旋转，旋转矩阵卡在 Q、K 之间且依赖位置，导致两个投影矩阵无法合并——这正是 MLA 与 RoPE 的冲突根源。详见上一章 RoPE 两个推导。

加 RoPE：解耦 RoPE（Decoupled RoPE）

冲突根源：RoPE 卡死了「权重吸收」

先把 MLA 推理省显存的真正机制说清——是权重吸收，不是别的。

朴素版里算分数是 q · k = q·(W_kb · c)。利用矩阵乘法结合律把括号挪一下：

q⊤(Wkb c)=(q⊤Wkb) cq^\top (W_{kb}\, c) = (q^\top W_{kb})\, c q⊤(Wkb​c)=(q⊤Wkb​)c

右边的含义：上投影矩阵 W_kb 不再去还原大 K，而是提前并入 q 一侧，先算出 q̃ = q·W_kb（投到 latent 空间），再直接和缓存的小 latent c 点积。全程不还原大 K——这就是吸收。

吸收为什么省？关键在 decode：一个 q 要对所有历史 s（共 t 个）算分数。

现在把 RoPE 加进来。RoPE 在 q、k 之间插一个依赖位置的旋转矩阵 R：

q⊤Rt−s Wkb csq^\top R_{t-s}\, W_{kb}\, c_s q⊤Rt−s​Wkb​cs​

R_{t-s} 卡在 q 和 W_kb 中间，且依赖相对位置 t−s（每对 token 都不同），所以 q·W_kb 再也合并不成一个固定矩阵——吸收失效。这就是 MLA 与 RoPE 的冲突：不在别处，正卡在权重吸收这个动作上。

解法：q、k 各劈成 nope + rope 两半

谁挡了吸收就把谁隔离。q、k 各拆成两部分，分数 = 两半各自点积再相加：

score=qC⋅kC⏟内容, 无RoPE, 可吸收+qR⋅kR⏟位置, 带RoPE, 单独算\text{score} = \underbrace{q^{C}\cdot k^{C}}_{\text{内容, 无RoPE, 可吸收}} + \underbrace{q^{R}\cdot k^{R}}_{\text{位置, 带RoPE, 单独算}} score=内容, 无RoPE, 可吸收qC⋅kC​​+位置, 带RoPE, 单独算qR⋅kR​​

内容半（nope）没有 RoPE 夹在中间，W_kb 照常吸收、省显存机制原样保留；位置半（rope）扛 RoPE，但维度小（qk_rope_head_dim，如 64），单独算代价可接受。

代码：推理（decode）吸收版

直接写推理吸收版：内容半在 latent 空间吸收、全程不升维 K/V，位置半带 RoPE 单独算。延续朴素版命名（w 前缀、RMSNorm），但把上投影拆成 wk_b / wv_b 两个——拆开纯为讲清吸收方向（wk_b 吸进 q、wv_b 吸进 wo），与朴素版合并写的 wkv_b 等价。

def apply_rope(x, cos, sin):
    # x: (B, H, L, d_h)；cos/sin: (1,1,L,d_h//2)，按位置预算好，可广播
    x1, x2 = x[..., 0::2], x[..., 1::2]          # 相邻两维拆成一对的两半
    rot_x1 = x1 * cos - x2 * sin                 # 二维旋转公式
    rot_x2 = x1 * sin + x2 * cos
    return torch.stack([rot_x1, rot_x2], dim=-1).flatten(-2)  # 交错铺回 d_h


class MLA_RoPE(nn.Module):
    """
    推理(decode)吸收版 MLA + 解耦 RoPE
    核心：cache 只存小 latent；内容部分权重吸收（不升维 K/V），位置部分带 RoPE 单独算
    """
    def __init__(self, dim, n_heads, qk_nope_head_dim, qk_rope_head_dim,
                 v_head_dim, kv_lora_rank):
        super().__init__()
        self.n_heads = n_heads
        self.kv_lora_rank = kv_lora_rank
        self.qk_nope_head_dim = qk_nope_head_dim          # 内容维
        self.qk_rope_head_dim = qk_rope_head_dim          # RoPE 维
        self.v_head_dim = v_head_dim                      # V 维, 不需要 rope, 保持内容维度对齐就行
        self.qk_head_dim = qk_nope_head_dim + qk_rope_head_dim   # nope + rope 拼起来

        self.wq = nn.Linear(dim, n_heads * self.qk_head_dim, bias=False)
        # KV 下投影：一次出 [要缓存的 latent | k_rope]
        self.wkv_a = nn.Linear(dim, kv_lora_rank + qk_rope_head_dim, bias=False)
        # 上投影拆两个，吸收版里不直接调用，只取 .weight 做预乘
        self.wk_b = nn.Linear(kv_lora_rank, n_heads * qk_nope_head_dim, bias=False)
        self.wv_b = nn.Linear(kv_lora_rank, n_heads * v_head_dim, bias=False)
        self.wo = nn.Linear(n_heads * v_head_dim, dim, bias=False)
        # wv_b 与 wo 预乘合并（推理准备阶段一次性算好，此处省略预乘逻辑）
        # self.wvo = (wv_b 升维 ⊗ wo) → (n_heads*kv_lora_rank, dim)

    def forward(self, x, cos, sin, kv_cache=None):
        B, L, _ = x.shape
        H, d_c = self.n_heads, self.kv_lora_rank

        # ===== Query：投影 -> 分头 -> 拆 [内容 | 位置] =====
        q = self.wq(x).view(B, L, H, self.qk_head_dim).transpose(1, 2)
        #  q (B, H, L, qk_head_dim)
        q_nope, q_rope = torch.split(
            q, [self.qk_nope_head_dim, self.qk_rope_head_dim], dim=-1)
        #  q_nope (B, H, L, qk_nope_head_dim)   q_rope (B, H, L, qk_rope_head_dim)

        # ===== KV 下投影：只对当前 x 算 latent 和 k_rope =====
        compressed = self.wkv_a(x)                                       # (B, L, kv_lora_rank + qk_rope_head_dim)
        c_kv, k_rope = torch.split(
            compressed, [self.kv_lora_rank, self.qk_rope_head_dim], dim=-1)
        #  c_kv (B, L, kv_lora_rank) <- 进 cache
        k_rope = k_rope.view(B, L, 1, self.qk_rope_head_dim).transpose(1, 2)
        #  k_rope (B, 1, L, qk_rope_head_dim) 头维=1, 全头共享

        # ===== KV Cache：拼历史，存的全是小 latent + 小 k_rope =====
        if kv_cache is not None:
            c_kv_past, k_rope_past = kv_cache
            c_kv = torch.cat([c_kv_past, c_kv], dim=1)                   # (B, kv_len, kv_lora_rank)
            k_rope = torch.cat([k_rope_past, k_rope], dim=2)             # (B, 1, kv_len, qk_rope_head_dim)
        new_kv_cache = (c_kv, k_rope)

        # ===== 内容部分：权重吸收，全程不升维 K =====
        # wk_b: (kv_lora_rank, H * qk_nope_head_dim) - (H * qk_nope_head_dim, kv_lora_rank) - 分头
        # qnope:bhl nopeheaddim * bh nopeheaddim lora - bhl lora
        # bhl lora * 
        W_kb = self.wk_b.weight.transpose(0, 1).view(d_c, H, self.qk_nope_head_dim)
        q_absorbed = torch.matmul(q_nope, W_kb)                          # (B, H, L, kv_lora_rank)
        latent_t = c_kv.transpose(1, 2).unsqueeze(1)                     # (B, 1, kv_lora_rank, kv_len)
        attn_nope = torch.matmul(q_absorbed, latent_t)                   # (B, H, L, kv_len)

        # ===== 位置部分：带 RoPE，老实算（维度只有 qk_rope_head_dim）=====
        q_rope = apply_rope(q_rope, cos, sin)                            # q: 当前位置
        k_rope = apply_rope(k_rope, cos, sin)                            # k: 含历史, 全序列位置
        # k_rope 头维=1，matmul 自动广播到 H（这就是「全头共享」）
        attn_rope = torch.matmul(q_rope, k_rope.transpose(2, 3))         # (B, H, L, kv_len)

        # ===== 两部分相加 = 注意力分数 =====
        attn = (attn_nope + attn_rope) / math.sqrt(self.qk_head_dim)
        attn = F.softmax(attn, dim=-1)

        # ===== 输出：V 也吸收，停在 latent 空间，不升维 V =====
        latent_v = c_kv.unsqueeze(1)                                     # (B, 1, kv_len, kv_lora_rank)
        ctx = torch.matmul(attn, latent_v)                               # (B, H, L, kv_lora_rank)
        ctx = ctx.transpose(1, 2).reshape(B, L, H * d_c)
        out = ctx @ self.wvo                                             # 预乘(wv_b·wo)一步回 hidden
        return out, new_kv_cache

易错点

  1. MLA 与 RoPE 的冲突点就在「权重吸收」：朴素版靠结合律 (q·W_kb)·c 把上投影并入 q、不还原大 K。RoPE 的旋转矩阵 R_{t-s} 卡在 q 和 W_kb 之间、且依赖相对位置 t−s，使两者无法预先合并成固定矩阵——吸收失效。不写吸收版，就讲不出解耦 RoPE 解决的到底是什么。
  2. 权重吸收省的是 decode：一个 q 对多个历史 k，大矩阵 W_kb 从「乘 t 次」降到「乘 1 次」，其余都是 q̃ 与小 latent 的廉价点积。prefill 时 q 是矩阵，吸收得对每行做，优势不明显。
  3. cache 里存的是小 latent c_kv（512）+ 一份 k_rope（64），不是升维后的大 K/V：若误存升维后的 k_nope/v，等于退回普通 MHA，KV Cache 一点没省。升维是每步现算、算完即弃的临时量，永不进 cache。
  4. 吸收版里 wk_b/wv_b 不再被当 nn.Linear 调用，只取 .weight 做预乘（q_nope @ W_kb）。这是「升维矩阵从一个运算步骤变成被吃进别处的一块权重」在代码里的样子——也是吸收版和朴素版 forward 的根本区别（不是参数不同，是算法不同）。
  5. k_rope 全头共享，只缓存一份：wkv_a 输出里 rope 段维度是 qk_rope_head_dim（单头量），reshape 成头维=1，靠 matmul 广播覆盖所有 head；k_nope 才是每头一份。
  6. q_rope 从 q 投影来（每头不同），k_rope 绕开 latent 直接从 x 投（全头共享）：若 k_rope 也从 c_kv 投，RoPE 会重新夹进压缩通路、冲突复现，故特意让它独立成路。
  7. apply_rope 的配对方式必须和 cos/sin 的生成方式严格对应：这里用 0::2/1::2 相邻配对（NeoX/DeepSeek 风格），cos/sin 必须按同一约定预算；若 cos/sin 按「前一半/后一半」（LLaMA-HF 风格）生成则会错位。配对约定是一对必须配套的东西。
  8. wvo 是预乘量：由 wv_b 与 wo 在推理准备阶段合并而成（代码省略预乘逻辑），面试要能说清它怎么来的——这是 V 侧吸收的落点。

训练版 vs 吸收版（口述要点，非手撕主体）

  • 训练时不能吸收：W_kb 要作为独立可求梯度的参数存在，一旦预乘进 q 就不再是独立参数，没法反向更新。所以训练用朴素版（老实升维），吸收是训练完、权重冻结后的一次性离线预处理（重参数化/预乘），不改变数值含义、也不是「换初始化」。
  • 手撕主体若想更短，可写朴素的解耦版（wk_b(c_kv) 老实升维再 split nope/rope），吸收部分口述；本笔记按"吸收能体现在代码里"的目标写了吸收版。

维度备忘（DeepSeek-V2）：kv_lora_rank=512、qk_nope_head_dim=128、qk_rope_head_dim=64、v_head_dim=128、n_heads=128。每 token 入 cache 仅 512+64=576 维，对比普通 MHA 的 n_heads·d_head·2 ≈ 32768 维。
