Hot 100 手撕思路

算法 手撕 leetcode hot100
手撕 LeetCode
Publish Date:   2026-05-30

按题型归类的 Hot 100 思路速记。只记思路和易错点，不写代码。带 ⚠️ 的是当时踩过的坑。

Backtracking

回溯的目标是剪枝，不能剪枝的回溯跟暴力遍历没区别

  • Letter Combinations of a Phone Number
    2-9 数字映射字母，列出所有组合。
    字典存映射 + 遍历字符串 + 字符串回溯。
  • Generate Parentheses
    列出所有合法括号组合。
    字符串回溯 + 两条规则：① 左括号没用完就无脑加；② 右括号数量比左括号少就无脑加。
  • Combination Sum
  • Permutations
  • N-Queens
  • Subsets
  • Word Search
  • Palindrome Partitioning
    返回所有可能的子集，切割之后剩余的部分都是回文数 即为有效子集

二分查找

  • Search in Rotated Sorted Array —— 在旋转数组中找目标值的 index。先找有序子数组 → 用有序性判断目标落在哪一侧 → 决定二分范围。

  • Search Insert Position
    有就返回 index，没有就返回插入位置。标准二分。
    ⚠️ 关键是返回 left 还是 right。

  • Median of Two Sorted Arrays

  • Find First and Last Position of Element in Sorted Array

  • Search a 2D Matrix

  • Binary Tree Maximum Path Sum

  • Find Minimum in Rotated Sorted Array

二叉树

  • Binary Tree Inorder Traversal
    DFS。
    ⚠️ 先写核心 DFS 逻辑判断，再补终止条件。
  • Validate Binary Search Tree ——
    设计左右区间，遍历时往下传区间值，每个节点判断是否落在区间内，有一个不符合就 false。
  • Symmetric Tree
    层序遍历 + 回文数组判断
    ⚠️ 要把 None 加进layer数组 而不是队列，加入队列会陷入死循环，你只需要判断当前层是否是对称（回文）就够了，不需要知道当前层的当前节点到底属于上一层的左节点还是右节点 这不重要，因为如果上一层是对称的，那么这一层的节点位置肯定也是对称的 如果不是的话 那么上一层肯定不是对称的 那就不会有后面层的问题了
  • Binary Tree Level Order Traversal
    层序遍历，按层输出。
  • Maximum Depth of Binary Tree —— 后序遍历。当前节点深度 = 自身这 1 层 + max(左子树深度, 右子树深度)。
  • Construct Binary Tree from Preorder and Inorder —— 返回根节点。前序定根 + 中序划分左右"地盘"。
  • Convert Sorted Array to BST —— 要深度平衡 → 取中点作根 + 前序构造。
  • Kth Smallest Element in a BST —— BST 中序遍历天然得到从小到大排序，取第 K 个。⚠️ 函数套函数时，子函数不能修改父函数的变量内容，但能访问/读取。
  • Path Sum III —— 根节点 + 目标值，统计有多少条从上到下（不分叉）的路径和 = 目标。前缀和 + 哈希表 + 回溯。核心：若 (当前前缀和 − 目标值) 在哈希表中，就找到一条合法路径；前缀和保证路径可以从任意节点起、任意节点止。
  • Flatten Binary Tree to Linked List
  • Binary Tree Right Side View
  • Invert Binary Tree
  • Lowest Common Ancestor of a Binary Tree
  • Diameter of Binary Tree

DP

  • Longest Palindromic Substring
    题目：输入字符串 输出最长回文子串
    DP[i][j] 数组从i到j组成的子串是否为回文子串 true or false
    递归：两侧相等 里面是回文 我就是回文

  • Longest Valid Parentheses
    题目：输入只有小括号组成的字符串 输出最长有效字串 即左右括号一一对应
    有效括号特性：它一定是连续的,而且结尾必须是右括号
    dp[i] = 以 i 结尾的最长有效长度
    递归：前面有效 我就有可能有效

  • Unique Paths
    题目：输入地图（矩阵）维度，输出从左上角都右下角 所有可能的路径的总数量
    dp[i][j] = 走到第 i 行第 j 列的不同路径数
    递推： dp[i][j] = dp[i-1][j] + dp[i][j-1],从上面来或从左边来两种情况相加。

  • Minimum Path Sum

  • Climbing Stairs

  • Edit Distance
    w1 → w2 的最小编辑距离（增删改）。dp[i][j] = 前 i 个到前 j 个字符所需最小操作数。
    ⚠️ w1 删 ≡ w2 增。

  • House Robber
    不能连抢两家，求最大金额。dp[i] = 前 i 间房能偷到的最大金额：不偷第 i 间 = dp[i-1]；偷第 i 间 = dp[i-2] + nums[i]
    ⚠️ 二维状态也能做（多一个状态），但一维更简单。

  • House Robber II
    房子排成环形
    思路同系列 1，环形拆成两种情况（去掉首 / 去掉尾）各跑一遍取 max。

  • House Robber III
    房子排成树，直连节点不能连偷。DFS 后序 + 每个节点"抢/不抢"两个状态。

  • House Robber V
    加颜色概念，相同颜色不能连偷，不同颜色可以。在系列 1 上加一个颜色判断，只影响"偷"的状态，"不偷"不受影响。

  • Best Time to Buy and Sell Stock 1
    只能买卖一次
    dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i]) # 不持股：保持，或今天卖
    dp[i][1] = max(dp[i-1][1], -prices[i]) # 持股：保持，或今天买（第一次买）

  • Best Time to Buy and Sell Stock 2
    无限次买卖
    dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i]) # 完全一样
    dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i]) # 持股：保持，或今天买

  • Best Time to Buy and Sell Stock 3
    最多两次。DP 四个状态（第 i 次持股 / 第 i 次不持股）。

  • Best Time to Buy and Sell Stock 4
    最多 k 次。系列 3 外面套一层 for 循环。

  • Best Time … with Cooldown
    无限次 + 冷冻期（系列 2 + 冷冻期）。在"不持股"状态里再分"处于 / 不处于冷冻期"。

  • Best Time … with Transaction Fee
    无限次 + 手续费（系列 2 + 手续费）。系列 2 里多减一个手续费。

  • Pascal’s Triangle

  • Word Break

  • Maximum Product Subarray

  • Coin Change II

  • Target Sum

  • Longest Common Subsequence

  • Longest Common Substring

图论

  • Number of Islands —— 统计被海洋隔开的岛屿数。DFS + 沉岛。⚠️ 时间复杂度恒为 R×C，但空间复杂度可优化。
  • Max Area of Island —— 最大岛屿面积。DFS + 沉岛 + 维护最大值。
  • Course Schedule
  • Rotting Oranges

贪心

  • Jump Game II —— 跳到最后一个 index 最少需要几步。核心：能不跳就不跳，到了当前能覆盖的边界值时才跳，这样步数一定最少。
  • Jump Game —— 能否从第一个跳到最后一个。贪心扩充：每一步更新能到达的最远距离，能到 → True，到不了 → False。
  • Partition Labels

哈希

  • Two Sum
  • Group Anagrams
  • Longest Consecutive Sequence
  • Subarray Sum Equals K

堆

  • Kth Largest Element in an Array —— 维护 k 个元素的最小堆，遍历数组和堆顶比较，最后堆顶即第 k 大。可用 Python 的 heapq（封装了堆操作）。
  • Top K Frequent Elements —— 返回 k 个高频元素。Hash + Heap：先统计频率，再按频率维护大小为 k 的最小堆，遍历堆返回所有值。⚠️ 堆相比暴力解的优势：只需维护 k 个数，暴力需全量求解。
  • Find Median from Data Stream

链表

  • Reverse Nodes in k-Group —— 每 k 个节点反转一次，剩余不足的保留。探测 + 反转链表 + 递归。⚠️ 递归本质 = 找到下一组的头。
  • LRU Cache
    最近最少使用：露头就 C 位。get = 查+挪；put = 查+(改)+挪 / 查+增+（删）+挪。
    方法：双链表（记顺序）+ 哈希表（搜索 O(1)）+ 两个哨兵处理头尾
  • Reverse Linked List —— 双指针 + temp 指针。⚠️ 一般不会用三指针，能双指针就双指针。
  • Add Two Numbers
  • Remove Nth Node From End of List
  • Merge Two Sorted Lists
  • Merge k Sorted Lists
  • Swap Nodes in Pairs
  • Copy List with Random Pointer
  • Linked List Cycle / Linked List Cycle II
  • Sort List
  • Intersection of Two Linked Lists
  • Palindrome Linked List

矩阵

  • Rotate Image —— 方阵原地旋转 90°。顺时针旋转 = 逆时针替换覆盖；旋转一圈后收缩到内圈，再下一轮。⚠️ 一定要逆时针赋值，这样只需存一次 temp。
  • Spiral Matrix —— 不一定是方阵。贪吃蛇思路：撞到就移动边界，边界越界就 break。⚠️ 和 Rotate Image 完全不同，方阵和一般矩阵的思路不能混。
  • Set Matrix Zeroes
  • Search a 2D Matrix II

滑动窗口

  • Longest Substring Without Repeating Characters —— 滑动窗口（双指针）+ 哈希表。
  • Sliding Window Maximum —— 返回每个窗口内的最大值数组。单调队列维护最大值，始终返回队头。⚠️ 难点：怎么维护单调性？怎么处理队头随窗口滑出（滑动窗口特性）？
  • Minimum Window Substring
  • Find All Anagrams in a String

栈

  • Valid Parentheses —— 判断括号是否一一对应。Stack + dict。
  • Daily Temperatures —— 从当天起还要等几天才有更高温度。单调栈。⚠️ 不需要 hash 表，栈里存索引本身就是天然的 hash——别把思维局限住（同理 hash 的 value 也可以存 node 而非单纯的值）。
  • Largest Rectangle in Histogram
  • Min Stack
  • Decode String

双指针

  • Container With Most Water —— n 条线，两条组成容器，求最大面积。双指针。核心：矮的那根往高的方向移。
  • Trapping Rain Water —— 和上一题区别：那题假设只有两条线，这题假设所有线都在，问下雨能接多少水。前缀/后缀最大值 + 双指针。
  • 3Sum
  • Move Zeroes
